将宿主机的本地目录挂载到 executor 沙箱中，使 agent 能够读取和写入你的真实文件。

## 适用场景

- 希望直接操作项目文件，而非沙箱中的副本
- 需要让 agent 访问本地资源（数据文件、配置目录等）
- 任务完成后，希望更改直接保留在宿主机上

## 使用方法

1. 在任务编排器或聊天面板中打开**本地文件系统**对话框
2. 将模式从**沙箱**切换为**本地挂载**
3. 添加一个或多个目录条目：
   - **名称**：挂载的人类可读标签
   - **路径**：宿主机上的绝对路径（例如 `/home/user/my-project`）
   - **权限**：只读（`ro`）或读写（`rw`）
4. 提交任务 — agent 将在容器内通过 `/workspace/.poco-local/` 看到这些目录

## 架构

```
宿主机                         容器
┌──────────────┐                ┌──────────────────────────┐
│ /home/user/  │  bind mount    │ /workspace/              │
│   my-project/ │ ────────────► │   .poco-local/          │
│              │                │     my-project/          │
│ /tmp/        │ ────────────► │   .poco-local/tmp/       │
└──────────────┘                └──────────────────────────┘
```

- 挂载目录在容器内以 `/workspace/.poco-local/<mount-id>/` 路径出现
- `rw` 挂载中的更改会直接写入宿主机文件系统
- 挂载内容不会进入沙箱的 git 历史和工作区快照

## 可用性

本地目录挂载由 `DEPLOYMENT_MODE` 环境变量控制：

| 值      | 行为                 |
| ------- | -------------------- |
| `local` | 挂载功能启用（默认） |
| `cloud` | UI 中禁用挂载功能    |

在 backend 配置中设置 `DEPLOYMENT_MODE=cloud` 即可在云端部署中禁用此功能。交互式初始化脚本（`scripts/quickstart.sh`）会自动询问此设置。

## 限制

- 仅适用于自托管部署（需要直接访问宿主机文件系统进行 bind mount）
- 云端部署时不可用
- 只读挂载无法被 agent 写入
