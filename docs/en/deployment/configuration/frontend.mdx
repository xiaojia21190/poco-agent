The frontend now uses the Next.js **same-origin API proxy** (`/api/v1/* -> Backend`) by default, so backend endpoint can be configured at **runtime**.

## Runtime

- `BACKEND_URL`: backend base URL used by Next.js server side to proxy `/api/v1/*` (Docker Compose default: `http://backend:8000`; local dev: `http://localhost:8000`; legacy alias: `POCO_BACKEND_URL`)

Note: The following are still build-time variables and are inlined into artifacts (see build args in `docker/frontend/Dockerfile`).

- `NEXT_PUBLIC_SESSION_POLLING_INTERVAL`: session polling interval in milliseconds (default `2500`)
- `NEXT_PUBLIC_MESSAGE_POLLING_INTERVAL`: message polling interval in milliseconds (default `2500`)
